# ConfigPilot v1.0.19

本版本改进 Codex API key 的保存体验，避免用户输入完成后还必须依赖其它配置操作才能写入认证文件。

## 主要更新

- API key 输入完成并离开输入框后自动保存到 `~/.codex/auth.json`。
- 按 Enter 或点击“保存密钥”仍可立即保存。
- 点击“应用更改”时会兜底保存仍留在输入框中的 API key。
- 保持 key 写入与模型列表请求的异步线程安全，不逐字符写入密钥。

## 升级说明

ConfigPilot 已包含应用内更新器。Windows 用户可在应用内完成下载、静默安装和重启；macOS 用户检测到新版后会打开官方 Release 页面。

如果模型接口仍返回 HTTP 401，请确认 API key 属于当前 API 地址对应的服务；“密钥已设置”只表示本地认证文件存在非空 key，不代表服务端已验证通过。

## macOS 首次安装须知

当前 macOS 版本仅支持 Apple Silicon（M 系列芯片），不支持 Intel Mac。

维护者目前没有 Apple Developer Program 会员，因此 DMG 尚未使用 Developer ID 签名和 Apple 公证。macOS Gatekeeper 可能显示“ConfigPilot 已损坏，无法打开”，但这不代表 DMG 文件真的损坏。

请先把 `ConfigPilot` 拖入 `Applications`，再打开“终端”执行：

```bash
xattr -dr com.apple.quarantine "/Applications/ConfigPilot.app"
open "/Applications/ConfigPilot.app"
```
